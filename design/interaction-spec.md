## 1. 一句话：从「单卡」升级成「场景盲盒 + 记录 + AI 调整」

v0 是：弹一张牌 → 抽一张 → 翻面 → 5 分钟 → 完成。
v1 主流程改成：

```
选场景(chip) → 该场景发 6 张牌背(翻翻乐) → 点一张翻开(盲盒惊喜)
   → 看动作详情 → 开始 5 分钟 → Done / 放弃
   → 完成庆祝(激励语) 或 拒绝(问原因)
   → 全程写入「记录」；AI 阶段性根据记录调整下次抽卡
```

不变的核心假设没动：还是「等 agent 时被递一张卡，做 5 分钟」。只是把"抽什么"做得更合场景、把"做没做"记下来、让推荐会进化。

## 2. 数据结构变更（moves.json）—— 这是要你重点看的

**都是增量字段，老引擎忽略它们仍能按 goal 加权随机跑，不会崩。** 新增三块：

### 2.1 `contexts` —— 场景（新）
```json
"contexts": {
  "office": { "label": "🏢 办公室/工位", "max": { "space":1, "noise":0, "social":0, "posture":1 } },
  ...
}
```
场景 = 一组**约束上限**。

### 2.2 每个 move 新增 `tags` —— 四维约束（新）
```json
{ "zh":"靠墙静蹲", ..., "tags": { "space":1, "noise":0, "social":0, "posture":1 } }
```
四个维度都是 `0..2`，越小越克制：

| 维度 | 0 | 1 | 2 |
|---|---|---|---|
| `space` 空间 | 原地 | 小范围(一臂) | 需展开 |
| `noise` 声音 | 静音 | 轻微 | 可出声 |
| `social` 社死度 | 不易察觉 | 正常 | 随意(可夸张) |
| `posture` 姿势 | 坐着 | 要站 | 要趴/躺 |

### 2.3 抽卡过滤规则（场景 → 能抽的动作）
> **某动作的每个 tag 都 ≤ 当前场景的 max，才进入这个场景的牌池。**
> 即 `move.tags[d] <= context.max[d]` 对 space/noise/social/posture 四个 d 都成立。

这样一个动作自动出现在所有"装得下它"的场景里，不用为每个地点重写卡组；加新场景只在 `contexts` 里加一条即可。我已经验证过：office 12 / cafe 8 / home 21 / hallway 14 张可抽，每个场景都够发 6 张。

### 2.4 顺带挪进 json 的文案（你之前是写死的，现在归 Claire）
- `config.defaultContext` 默认场景、`config.drawCount` 一次发几张(6)
- `doneMessages[]` 完成激励语（随机取一条）
- `refuseReasons[]` 拒绝原因选项

## 3. 引擎要做/改的点（按优先级）

### P0（demo 必须有，建议本次做）
1. **读 `contexts` + `tags`，按场景过滤后再加权随机发 `drawCount` 张**（替换原来的全库随机）。当前目标加权 ×3 的逻辑保留，只是先过场景这道滤网。
2. **6 张翻翻乐**：发 6 张牌背 → 点一张翻开该张（其余变暗）→ 进入动作详情。原型里 `pick()` 是这个交互。
3. **场景 chip 切换**：顶部一排 chip，切换即重发 6 张。默认用 `config.defaultContext`。
4. **完成/拒绝两条出口**：
   - 完成 → 随机 `doneMessages` + 庆祝 + 通知宿主 `workout_done`（同 v0，让螃蟹比心）。
   - 拒绝 → 弹原因 chip（`refuseReasons`）→ 通知宿主 `workout_skipped`。

### P1（记录功能，要先对一下"存哪"，见第 4 节）
5. **记录每次结果**：`{动作, 场景, 完成/拒绝, 拒绝原因, 时间}` 落地存储。
6. **记录页**：统计（本周完成 / 连续天数 / 完成率）+ 列表（可按 全部/完成/拒绝 筛选）。原型第 ⑥ 屏。

### P2（Agents，可后做，先占位）
7. **阶段性调整抽卡**：基于记录做**规则式**重加权即可，不必上模型——例如「某动作连拒 N 次→降权/暂时不发」「某场景只接受拉伸→该场景调高拉伸权重」「最常拒因是『正忙』→默认时长调短」。
8. 原型里抽卡页顶的 banner 和记录页的「AI 洞察」就是这个功能的 UI 出口，先静态文案占位，后面接真实统计。

## 4. 待定决策（需要你拍板 / 一起定）

1. **记录存哪？**（README 里你提过这是另一个量级的工程）
   我的建议：v1 只做**本地存储**（localStorage 或宿主侧一个本地 json 文件），不联网、不碰云、不收隐私敏感数据，只存动作名/场景/结果/原因/时间戳。够喂记录页和规则式 AI 了。你看在 clawd-on-desk 那侧用哪种更顺手。

2. **卡片结构**：翻翻乐 6 张是新结构，确认按原型的「点一张翻一张、其余变暗」来；如果你更想要「翻开还能反悔再翻别的」，告诉我，我调原型。

3. **浮窗尺寸**：6 张牌 + chip 比 v0 单卡大，原型面板宽 ~408px。注入 Electron 透明窗时尺寸要相应放大，这块归你。

4. **字体**：neobrutalism 版用了思源黑体（Noto Sans SC，走 Google Fonts CDN）。如果上线要完全离线，需要把字体打包进项目本地引用——这步等你定运行环境再说。

## 5. 边界（v1 仍不做）
- 不接屏幕使用时间统计。
- 记录只本地、不上云、不做账号。
- AI 先规则式，不引入训练/模型服务。

先把「场景盲盒 + 记录」跑通，验证「人会不会按场景动起来」，再加 AI 的深度。
